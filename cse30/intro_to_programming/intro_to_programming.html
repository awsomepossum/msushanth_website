README

	

Description:


This program takes a single integer as input from the command line and
prints a short message on the terminal that includes a welcome message, when
the assignment is due, and the square of the integer that was passed in.


	

How to Compile:


To compile this program, go to the directory containing the source files
for pa0 (which should also include the Makefile) and type "make".


	

How to Run:


 To run the program, type the executable name followed by an integer
argument. 
Example:
        ./a.out 7


	

Normal Output:


Normal output is printed to stdout. An example of normal output would be
the following:
        [cs30xxxx@ieng9]:pa0:208$ ./a.out 7
        Welcome to CSE 30, Spring 2016
        PA0 is due on April 5, 2016
        The square of 7 is 49


	

Abnormal Output:


Abnormal output is printed to stderr. This can occur if an integer is not
specified or if there are too many arguments.

When an integer is not specified:
        [cs30xxxx@ieng9]:pa0:210$ ./a.out  
        ./a.out: missing integer arg
        Usage: ./a.out [INTEGER]
         Integer value to square        

When there are too many arguments passed in:
        [cs30xxxx@ieng9]:pa0:211$ ./a.out 1 1
        ./a.out: too many arguments
        Usage: ./a.out [INTEGER]
         Integer value to square


	

Testing:


This program was tested using the sample pa0test that was provided. Both
programs were run with the same input and their outputs were inspected to see
if they were different.
Running this code:
        ./a.out 7 >& MYSOL
        ~/../public/pa0test 7 >& REFSOL
        diff -c MYSOL REFSOL
will redirect the output of a.out to MYSOL and pa0test to REFSOL. Then it
will compare the two output files using the diff command. If the two output
are the same, then it will indicate that there are no differences.

Furthermore, the squareNum() function was unit tested to ensure that it
returns the proper value. Test inputs include positive numbers, negative
numbers, 0, and strings (like "a" or "asdasd") to see if the two programs have
the same output.
