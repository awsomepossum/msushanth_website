Description:


This assignment further explores OpenMP, an API that supports
multi-platform chared memory multi-threading with minimal effort. We will
compare the runtime of the serial implementation and the parallel version.
The algorithms we are writing are numOfFactors. There will be a serial
version and a parallel version. The parallel version will be identical to the
corresponding serial version, but with added OpenMP pragmas. This program
will take a number as input and loop thru the range [lo, hi) and look to see
how many times that number is evenly divisible by the numbers in the range.


	

How to Compile:


To compile this program, go to the directory containing the source files
for pamt2 (which should also include the Makefile) and type "make".


	

How to Run:


To run this program, type the executable name followed by a number (or
group of numbers separated by spaces).
Example:
        [cs30xxxx@ieng6-201]:pamt2:517$ ./pamt2 1234567  
        [cs30xxxx@ieng6-201]:pamt2:518$ ./pamt2 1234567 7654321 13579 97531

Abnormal Output:


Abnormal output is printed to stderr. This can occur if a number being
passed in is too large or not a number.
        [cs30xanm@ieng6-201]:pamt2:538$ ./pamt2 99999999999999999999999999999

        Skipping 99999999999999999999999999999
        num should be less than or equal to 8589934592


        [cs30xanm@ieng6-201]:pamt2:539$ ./pamt2 99999999a                    

        Skipping 99999999a
        not a valid long long int


	

Testing:


This program was tested using the sample pamt2test executable that was
provided. Both programs were run with the same input and their outputs were
inspected to see if they were different.
Running this code:
        ~/../public/pamt2test 123 456 789 >& REFSOL
        ./pamt2 123 456 789 >& MYSOL
        diff -c MYSOL REFSOL
will redirect the output of pamt2test to REFSOL and pamt2 to MYSOL. Then we
will compare the two output files using the diff command. If the two output
are the same, then it will indicate that there are no differences. We should
ignore the time ti took to complete each method and the speed up times since
they will differ every time we run this program.
