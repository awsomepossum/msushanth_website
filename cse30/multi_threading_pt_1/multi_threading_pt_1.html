Multi-Threading & Debugging pt. 1

Email sushanthmukkamalla@gmail.com for code

README

	

Description:


This assignment will explore OpenMP, an API that supports multi-platform 
shared memory multi- threading with minimal effort. Serial code can be easily
made parallel by the addition of added directives (OpenMP pragmas) that will
instruct the compiler to generate parallel code. This program will use a
12-byte encoded pattern that needs to be decoded to initialize an array of a
user-specified length. Once the array is initialized correctly, the program
will perform two modified sum calculations and find the min and max values.
Each step of this program will be performed it two different ways:

1) using a serial implementation
2) with a parallel version.

We will then compare the runtime of each version.

The three algorithms we are writing are: initData, squaredSumMinMax, and
sqrtSumMinMax. For each of these algorithms, there will be a serial version and
a parallel version. Each of the parallel versions will be identical to the
corresponding serial version, but with added OpenMP pragmas.


	

How to Compile:


To compile this program, go to the directory containing the source files
for pamt1 (which should also include the Makefile) and type "make".


	

How to Run:


To run the program, type the executable name followed by an array size.
Example:
        [cs30xxxx@ieng6-203]:pamt1:128$ ./pamt1 99999

Abnormal Output:


Abnormal output is printed to stderr. This can occur if the first command
line argument is not in the range [12 - 4294967295] or not a number. For all
abnormal input, this is printed out:
        [cs30xxxx@ieng6-203]:pamt1:133$ ./pamt1 99999a               

        Usage: ./pamt1 array_size
            array_size  (must be >= 12 and <= 4294967295)


	

Testing:


This program was tested using the sample pamt1test executable that was
provided. Both programs were run with the same input and their outputs were
inspected to see if they were different.
Running this code:
        ~/../public/pamt1test 420 >& REFSOL
        ./pamt1 420 >& MYSOL
        diff -c REFSOL MYSOL
will redirect the output of pamt1test to REFSOL and pamt1 to MYSOL. Then it will
compare the two output files using the diff command. If the two outputs are
the same, then it will indicate there are no differences. We should ignore
the time it took to complete each method since the times will differ every
time we run this program.
