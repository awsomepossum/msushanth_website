Description:


The purpose of this programming assignment is to further emphasize C and
implement a real Unix command(along with using more Standard C Library
routines). I built a program called "myls" that will function very
similarly to the real Unix command "ls". This program will list information
about the files (or directories) entered as a command line argument. If
files/directories are not specified, "myls" prints information about the
current directory by default. Some of the system calls and Standard C Library
routines I used include getopt_long(), lstat(), opendir(), readdir(),
closedir(), malloc(), realloc(), free(), qsort(), getpwuid(), getgrgid(),
strcmp(), and ctime().


	

How to Compile:


To compile this program, go to the directory containing the source files
for pa3 (which should also include the Makefile) and type "make".


	

How to Run:


To run the program, type the executable name "myls" followed by zero or
more of these flags: -a, --all, -l, -r, --reverse, -t, -B, --ignore-backups,
-F, --classify, -h, --help. You can put the flags in any order. You can
combine the flags with one letter together into one flag. After the flags,
you can specify a directory, filename, or multiple of each. You can also
specify none to use the current directory. Examples:

        ./myls
        ./myls -a
        ./myls -l
        ./myls -r
        ./myls -t
        ./myls -B
        ./myls -F
        ./myls -h
        ./myls -alrt
        ./myls --all
        ./myls --reverse
        ./myls --ignore-backups
        ./myls --classify
        ./myls --help
        ./myls --reverse --ignore-backups --classify
        ./myls -alrt --ignore-backups
        ./myls ./test
        ./myls ./printPermissions.c
        ./myls -alrt ~/../public ~/../
          etc...

Testing:


This program was unit tested in parts using the testbuildFileInfoTable.c,
testmodTimeCompareRev.c, testnameCompare.c, and testparseArgs.c programs.
    
This program was also tested using the sample pa4ref that was provided.
Both programs were run with the same input and their outputs were inspected
to see if they were different.
Example of running this code:
 ~/../public/pa4ref -alrt ~/../public ~/../ ~/../public/pa4SampleFiles >& REFSOL
 ./myls -alrt ~/../public ~/../ ~/../public/pa4SampleFiles >& MYSOL
 diff -c REFSOL MYSOL
will redirect the output of pa4ref to REFSOL and myls to MYSOL. Then it will
compare the two output files using the diff command. If the two outputs are
the same, then it will indicate there are no differences. If errors were
encountered, it will show where.
