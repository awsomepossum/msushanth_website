README

	

Description:


This program draws an S. It takes in 4 inputs from the command line. The
first argument is the width of the S (it must be even and within the range of
4 and 1000). The second argument is the ascii value of the character making
up the S (must be within the range of 32 and 126). The third argument is the
ascii value of the character filling the space between the S and the border
(must be within the range of 32 and 126 and must be a different value than
the one making up the S). The fourth argument is the character making up the
rectangular border around the S (must be within the range of 32 and 126 and
must be a different vale than the one making up the S). With these arguments,
pa1.c's main() does error checking on each input passed in to check if they
are numbers, within the range, there are the correct number of arguments passed
in, ..., and prints the appropritate error messages to the screen. If there
are no errors, the drawS.s assebly module is called and an S is printed to
the screen. To do this, the drawS.s assembly file makes use of loops and
calls to the printSegment.s and printChar.s modules.


	

How to Compile:


To compile this program, go to the directory containing the source files
for pa1 (which should also include the Makefile) and type "make"


	

How to Run:


To run the program, type the executable name followed by four integer
arguments.
Example:
    ./pa1 8 45 63 35


	

Normal Output:


Normal output is printed to stdout. An example of normal output would be
the following:
        [cs30xxxx@ieng9]:pa1:519$ ./pa1 4 35 32 33
        !!!!!!
        ! ## !
        !#   !
        !#   !
        ! ## !
        !   #!
        !   #!
        ! ## !
        !!!!!!


	

Abnormal Output:


Abnormal output is printed to stderr. This can occur if any of the command
line inputs are out of their range, if the third or fourth arguments are the
same as the second one, the first argument is not even, if any of the inputs
are not numbers, if there are not exactly 4 arguments, or if a parameter is
an extremely large number.

When there are not exactly four arguments:
    [cs30xxxx@ieng9]:pa1:$511 ./pa1 4 32 32 32 32

    Usage: ./pa1 sWidth sChar fillerChar borderChar
        sWidth      (must be within the range of [4 - 1000])
                    (must be even)
        sChar       (must be an ASCII value within the range [32 - 126])
        fillerChar  (must be an ASCII value within the range [32 - 126])
                    (must be different than sChar)
        borderChar  (must be an ASCII value within the range [32 - 126])
                    (must be different than sChar)

When an argument is not a number:
    [cs30xxxx@ieng9]:pa1:$515 ./pa1 10a 33 34 35
        
            "10a" is not an integer

When a parameter is an extremely large number:
    [cs30xxxx@ieng9]:pa1:$521 ./pa1 99999999999999 33 34 35

            Converting "99999999999999" base "10": Result too large

When the inputs are out of their range:
    [cs30xxxx@ieng9]:pa1:$529 ./pa1 2 1 2 2

            sWidth(2) must be within the range of [4 - 1000]
    
            sChar(1) must be an ASCII code in the range [32 - 126]

            fillerChar(2) must be an ASCII code in the range [32 - 126]

            borderChar(2) must be an ASCII code in the range [32 - 126]

When the third or fourth arguments are tha same as the second argument:
    [cs30xxxx@ieng9]:pa1:$529 ./pa1 4 35 35 35

            sChar(35) and fillerChar(35) must be different

            sChar(35) and borderChar(35) must be different

When the first argument is not an even number:
    [cs30xxxx@ieng9]:pa1:$529 ./pa1 5 35 32 32
    
            sWidth(5) must be even


	

Testing:


The program was unit tested in parts usng the testisEven.c, testisInRange.c,
and testnumOfDigits.c files to test the assembly modules isEven.s, isInRange.s,
and printSegment.s. We wrote our own test in the test files to check if the
assembly modules worked properly.

    This program was also tested using the sample pa1test that was provided.
Both programs were run with the same input and their outputs were inspected
to see if they were different.
Running this code:
        ~/../public/pa1test 4 35 32 33 >& REFSOL
        ./pa1 4 35 32 33 >& MYSOL
        diff -c REFSOL MYSOL
will redirect the output of pa1test to REFSOL and pa1 to MYSOL. Then it will
compate the two output files using the diff command. If the two outputs are
the same, then it will indicate there are no differences. If the incorrect
number of command line arguments are passed in to the test and our program
and are compared using the diff command, the first line should be ignored
since the names of the executables are different.
