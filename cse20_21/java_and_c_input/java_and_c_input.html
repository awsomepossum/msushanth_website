Assignment Instructions

What to do?

In this assignment, you are asked to write functions that read strings and whole numbers from the user, processes them, and displays them. You will be writing these functions in the hw2.java file and the hw2.c file. Your programs should terminate when the user enters ^D (Ctrl-d). At termination, your hw2.c program should output the sizes of various C pre-defined variable types.



The functions to write in hw2.java:


      void clrbuf (int character);
      long decin ();
      long getaline (char message[], long maxlength);


The functions to write in hw2.c:


      void clrbuf (int character);
      long decin (void);
      long getaline (char * message, long maxlength);


The functions needed from your hw1:


      baseout, decout, newline, writeline


You must write debug statements in the following functions:


      getaline, writeline

Function Descriptions

	

clrbuf:


Removes any characters in the stdin buffer by repeatedly calling fgetc until a newline character is detected. The
method is implemented by calling fgetc until the buffer is empty. You know the buffer is empty when the last character
you removed was a a newline character '\n'.

* Values of character are expected to be the most recent character read from stdin.


	

decin:


Reads a decimal number from stdin and converts it from a sequence of ASCII characters into a decimal whole number.
This converted value is returned. When an overflow occurs or non-digit characters are entered, the user is prompted
to reenter the number using the digiterror function (provided).  The best way to check for overflow is after you
perform a math operation, perform the opposite operation to see if you get the original value back. As for non-digit
characters, just check if the character you grabbed using fgetc is outside the values of the '0' and '9' characters.
You should be using a loop to keep grabbing characters from stdin until you hit the newline character '\n'.  If it's
a valid character, convert it to a decimal number by subtracting the '0' character from it ( look at an ASCII table
to see why that works ).  Finally, to add this to the sum you're returning, you just use the algorithm of multiplying
your current sum by 10 and adding the converted digit so it falls in the one's place.  Don't forget to check for EOF!
And know that in hw2.c's getaline, you should store a null character, '\0', at the end of the string. The reason is
because the null character serves as a flag of when to stop when looping through an array in C.  It's unlike Java
where we can just use the array field length.


	

decout:


Displays number in base 10 to the filestream stream.

* Values of number are expected to be any positive whole number.
* Values of stream are expected to be either stdout or stderr.


	

getaline:


Reads characters from stdin or System.in and stores them into the message array. The number of characters stored into
message must not exceed maxlength (including NULL character). Any extra characters entered to stdin are discarded
using clrbuf. Add a debug statement which prints the number of characters stored in message. The function should
return the number of characters stored. Don't forget to check for EOF!

* Values of message are expected to be an array to hold the characters entered.
* Values of maxlength are expected to be the size of the message array.


	

hexout:


Displays number in hexadecimal with a preceding 0x. The number will always be printed with 8 digits (ex. 0x000ABC12,
0x00000000) when a long is 4 bytes and 16 (ex. 0x0123456789ABCDEF, 0x0000000000000000) digits when a long is 8 bytes.

* Values of number are expected to be any positive whole number.
* Values of stream are expected to be either stdout or stderr.


	

writeline:


In hw2.c, update the writeline code you wrote from hw1 by adding a debug statement to display the length of the string
printed. Add code to return the number of characters printed.

Implementation Restrictions

To implement these functions, you may not make calls to ANY functions you did not author except for fputc and fgetc. You may call on the String methods charAt and length in code originating from hw1.java. For more information about fputc and fgetc, look at the man page by typing man fputc or man fgetc into the command prompt or into Google. You can also use fprintf for error and debug messages using stderr , and you can use System.err.print or System.err.println for error and debug messages.
