Program Description:

  Rational.java:
    This program defines all the methods used to run the test cases in
  TestRational.java and EC_TestRational.java. When you create a Rational object
  you are actually creating a fraction. If you create a blank Rational object,
  you are actually creating 0/1/ If you create a Rational object passing in
  one number, you are actually creating (number passed in)/1. If you create a
  Rational object passing in two numbers, you are actually creating the
  fraction (num1)/(num2). If the denominator is ever 0, the program will throw
  an exception so that the program will not exit out and keep running. You can
  add, subtract, multiply, and divide the Rational objects fractions. The
  fractions will always be stored in their simplest, reduced form. The
  numerator can be negative or positive, but the denominator is always
  positive. You can also get the reciprocal of the number (if the numerator is
  not 0), round up the fraction into a whole number, or round down the fraction
  into a whole number.
