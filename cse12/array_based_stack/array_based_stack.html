Assignment Instructions

What to do?

In this assignment, you are to implement an array-based stack of longs and modify the driver.c file which allows you to test the stack.



The functions to write in stack.c:


      void delete_Stack (Stack ** spp);
      void empty_Stack (Stack * this_Stack);
      long isempty_Stack (Stack * this_Stack);
      long isfull_Stack (Stack * this_Stack);
      Stack * new_Stack (unsigned long stacksize);
      long num_elements (Stack * this_Stack);
      long pop (Stack * this_Stack, long * item);
      long push (Stack * this_Stack, long item);
      long top (Stack * this_Stack, long * item);


Add the following to driver.c:


      Add switch cases for 'a' (allocate stack), 'd' (deallocate stack), and 'e' (empty stack).
      In a comment block, answer the debugger questions found near the end of the Development Guide.

Stack Infrastructure

Your array-based stack will be dynamically allocated from the heap. It will have the following structure:
             ———————————————————————————————
             | Stack Count | Stack Size | Stack Pointer |      |      |   ...
             ———————————————————————————————
   index:           -3                  -2                     -1              0       1      ...

      * The Stack Count is the stack number. The first stack created will have number 1, the second stack created          will have number 2, etc.
      * The Stack Size is the capacity of the stack. This is determined by the value stacksize passed into          new_Stack when the stack is created.
      * The Stack Pointer holds the index of the last used space in the stack. For empty stacks this would be -1.

Function Descriptions

	

delete_Stack:


This destructor function deallocates all memory associated with the stack and sets its pointer in the calling function
to NULL.

* Values of spp are expected to be a double pointer to the stack that is to be deleted.


	

empty_Stack:


Effectively empties the stack of all its elements.

* Values of this_Stack are expected to be a pointer to the stack to be emptied.


	

isempty_Stack:


Returns true if this_Stack is empty, and false if it is not.

* Values of this_Stack are expected to be a pointer to the stack to be checked.


	

isfull_Stack:


Returns true if this_Stack is full, and false if it is not.

* Values of this_Stack are expected to be a pointer to the stack to be checked.


	

new_Stack:


This constructor function allocates and initializes a new Stack object. It allocates memory to hold stacksize number
of longs, initializes the stack infrastructure, and returns a pointer to the first storage space in the stack.

* Values of stacksize are expected to be positive integers.


	

num_elements:


Returns the number of elements in this_Stack.

* Values of this_Stack are expected to be a pointer to the stack to be checked.


	

pop:


Removes an item from the top of the stack, and sends it back through the output parameter item. Returns true upon
success and false upon failure.

* Values of this_Stack are expected to be a pointer to the stack from which to pop.
* Values of item are expected to be a pointer to a long which will store the popped item.


	

push:


Adds item to the top of this_Stack.

* Values of this_Stack are expected to be a pointer to the stack on which to push.
* Values of item are expected to be a long to be pushed on the stack.


	

top:


Sends back the item on the top of the stack through the output parameter item, but does not remove it from the stack.
Returns true upon success and false upon failure.

* Values of this_Stack are expected to be a pointer to the stack from which to top.
* Values of item are expected to be a pointer to a long which will store the topped item.
