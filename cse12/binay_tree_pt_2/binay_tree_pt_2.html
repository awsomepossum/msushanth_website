Assignment Instructions

What to do?

In this assignment, you are to implement a binary tree data structure to use the "template" feature of C++.

Your Tree.c code will implement a binary tree as did Tree.java before but will implement a proper remove as well as a threshold check to maintain the appropriate balance of the tree.

Your Driver program will work like the sample driver program on-line. You will need to complete the definition for the UCSDStudent class in your Driver.h.



The TNode methods to write in Tree.c:


      TNode (const Whatever & element, Tree<Whatever> & theTree)
      TNode (const Whatever & element, TNode<Whatever> & parentTNode)
      ~TNode (void)
      void delete_AllTNodes (void)
      void ReplaceAndRemoveMin (TNode<Whatever> & targetTNode, TNode<Whatever> *& PointerInParent)
      unsigned long Remove (TNode<Whatever> & elementTNode, TNode<Whatever> *& PointerInParent,
                 long fromSHB = FALSE)
      void SetHeightAndBalance (TNode<Whatever> *& PointerInParent)
      unsigned long Insert (const Whatever & element, TNode<Whatever> *& PointerInParent)


The Tree methods to write in Tree.c:


      unsigned long Insert (const Whatever & element)
      unsigned long Lookup (Whatever & element) const
      unsigned long Remove (Whatever & element)
      ~Tree (void)


Code to write in Driver.h:


      Add appropriate public class methods for UCSDStudent

Function Descriptions

Note: The non-constant reference parameters (noted in blue in Function Descriptions) are potentially updated within the body of that method. 
	

TNode Constructor (for the root TNode):


Called when creating the root TNode.

* Values of element are expected to be the data to be stored in the TNode. They must be of the same type as the rest
  of the objects present in the tree.
* Values of theTree are expected to be a reference to the empty Tree where the root TNode is about to be created.


	

TNode Constructor (for all other TNodes):


Called when creating any other TNodes.

* Values of element are expected to be the data to be stored in the TNode. They must be of the same type as the rest
  of the objects present in the tree.
* Values of parentTNode are expected to be a reference to the parent of the TNode about to be created.


	

TNode Destructor:


Called when deallocating a TNode.


	

delete_AllTNodes:


Performs a post-order traversal through the Tree deleting each TNode.


	

ReplaceAndRemoveMin:


Called when removing a TNode with 2 children, replaces that TNode with the minimum TNode in its right subtree to
maintain the Tree structure.

* Values of targetTNode are expected to be a reference to the TNode to remove that has 2 children.
* Values of PointerInParent are expected to a reference to the TNode pointer in the parent TNode used to get to the
  current TNode.


	

TNode's Remove:


Removes the matching data from the binary tree. Returns true or false indicating success of removal. 

* Values of elementTNode are expected to be a reference to the TNode containing the data that identifies the element
  to remove.
* Values of PointerInParent are expected to a reference to the TNode pointer in the parent TNode used to get to the
  current TNode.
* Values of fromSHB are expected to be true or false, keeping track of whether or not Remove was called from
  SetHeightAndBalance so that Remove can determine whether or not to call SetHeightAndBalance.

	

SetHeightAndBalance:


Updates the height and balance of the current TNode.

* Values of PointerInParent are expected to be a reference to the TNode pointer in the parent TNode used to get to
  the current TNode.


	

TNode's Insert:


Inserts an element into the binary tree. Returns true or false indicating success of insertion. 

* Values of element are expected to be the data to be stored in the TNode. They must be of the same type as the rest
  of the objects present in the tree.
* Values of PointerInParent are expected to be a reference to the TNode pointer in the parent TNode used to get to
  the current TNode.


	

Tree's Insert:


Inserts an element into the binary tree. Inserts at the root TNode if Tree is empty, otherwise delegates to TNode's
Insert. Returns true or false indicating success of insertion. 

* Values of element are expected to be the data stored in the TNode. They must be of the same type as the rest of the
  objects present in the tree.


	

Tree's Lookup:


Searches for an element in the Tree. Delegates to TNode's Lookup when implemented recursively. Returns true or false
indicating success of look up. 

* Values of element are expected to be the data stored in the TNode. They must be of the same type as the rest of
  the objects present in the tree.


	

Tree's Remove:


Removes an element from the Tree. Delegates to TNode's Remove when Tree is not empty. Returns true or false indicating
success of removal. 

* Values of element are expected to be the data stored in the TNode. They must be of the same type as the rest of
  the objects present in the tree.


	

Tree Destructor:


Deallocates memory associated with the Tree.

Implementation Restrictions

* All data stored within the SymTab (and thus stored in the Tree and TNode) must be "flat", which means that    there are no pointer fields within objects of classes to go into the SymTab.
* TNode's Insert and TNode's Remove should be implemented via recursion.
* TNode's Lookup can be either loop-based or recursive.
* "structs" are not to be defined in .h files - all should be "classes".
* Public access to data fields in classes defined in the .h files are not allowed.
