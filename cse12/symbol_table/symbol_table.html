Assignment Instructions

What to do?

In the assignment, you are to implement a Symbol Table to allow variable assignment to the calculator. You are also to write the member functions for the UCSDStudent class as part of the Driver.java file and the Driver.c file used to test your symbol table.

Modifications to provided and existing code:
      You are to provide the missing code to the member functions in the HashTable.java file and the Hash.c file,
      and the missing portions of the UCSDStudent class definition to the Driver.java file and the Driver.c file.
      The Hashing algorithm will be double hashing using a bully algorithm to decide between conflicts as
      discussed in class.



The functions to write in HashTable.java:


      public static void setDebugOn ()
      public static void setDebugOff ()
      public HashTable (int sz)
      private boolean insert (Base element, boolean recursiveCall)
      private Base locate (Base element)
      public Base lookup (Base element)


The functions to write in Hash.c:


      void HashTable :: Set_Debug (long option)
      HashTable :: HashTable (long sz) : size (sz), table_count(++counter), occupancy (0), table (new Base *[sz])
      HashTable :: ~HashTable (void)
      long HashTable :: Insert (Base * element, long recursiveCall)
      const Base * HashTable :: Locate (const Base * element) const
      const Base * HashTable :: Lookup (const Base * element) const


Additional code to write in Driver.java and Driver.c:


      Add appropriate member functions for UCSDStudent

Function Descriptions

	

set_debug_on:


Turns on debugging for this list.


	

set_debug_off:


Turns off debugging for this list.


	

HashTable Constructor:


Allocates and initializes the memory associated with a hash table.

* Values of sz are expected to be any positive prime. (You do not have to check if it is prime)


	

HashTable Destructor (only in C++):


Called when de-allocating this HashTable


	

insert:


Inserts the element in the hash table. Returns true or false indicating success of insertion.

* Values of element are expected to be complete elements to insert.
* Values of recursiveCall represent whether or not this function was called recursively. The value is expected to be
  false when called from Driver's main.


	

locate:


Locates the index in the table where the insertion is to be performed, an item is found, or an item is determined not
to be there. Sets the variable index to the last location checked; it will be used by insert and lookup. Returns the
item if an item with matching name is found. Otherwise, return null.  Loops stops when encountering a null table
location or when encountering an item that is smaller than the parameter item.

* Values of element are expected to be complete and incomplete elements depending on whether it's called from insert
  or lookup.


	

lookup:


Looks up the element in the hash table. Returns pointer to the element if found, null otherwise.

* Values of element are expected to be complete and incomplete elements depending on whether it's called from insert
  or lookup.
