Assignment Instructions

What to do?

In this assignment, you are to develop a program that will simulate a top-of-the-line calculator that will perform such advanced commands as addition, subtractions, multiplication, division, exponentiation, and factorial! The program will accept mathematical expressions in "infix" notation such as...


      ( ( 27 + 452 ) * 2 - 16 ^ 2 ) / 7


convert it to its "postfix" equivalent:


      27 452 + 2 * 16 2 ^ - 7 /


and then evaluate this expression. All operations are integer operations. Don't worry about negative input.



The functions to write in calc.c:


      long eval (Stack * stack1);
      long intopost (Stack * stack1);
      static long exponent (long power, long base);
      static long fact (long xxx, long ignored);
      static long setupword (int character);


The files needed from your hw3:


      stack.c

Function Descriptions

	

eval:


Utilizing 2 Stacks, evaluate mathematical expressions from "postfix" notation. Refer to the calc.c.empty for the
algorthm.

* Values of stack1 /* input parameter */ are expected to be pointers to Stacks containing "postfix" expressions to 
  evaluate.


	

intopost:


Utilizing 2 Stacks, convert "infix" mathematical expressions entered by the user into their "postfix" equivalents.
Refer to the calc.c.empty for the algorthm.

* Values of stack1 /* output parameter */ are expected to be pointers to empty Stacks used to store "postfix"
  expressions.


	

exponent:


Raising base to the power exponent.

* Values of base are expected to be any positive integer.
* Values of power are expected to be any positive integer.


	

fact:


Calculating xxx factorial.

* Values of xxx are expected to be any positive integer.
* Values of ignored can be any valid integer.


	

setupword:


Constructor funtion for longs representing operators to be stored on your Stacks. The representation for the operators
should have everything associated with that operator: a distinction from numbers, the index in the functions array
corresponding to that operator, and the priority of the operator and the ASCII code.

* Values of character are expected to be ASCII codes for operators entered by the user, which are also expected to be
  found inside the operators[] array.
